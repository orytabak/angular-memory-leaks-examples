Leaking observable component
Subscribing to an Observable is a useful way to get your data. But when the subscription is made outside of the root scope, it will probably cause memory leaks.
The reason is that the Observalbe, even if declared inside a certain scope, will remain in memory as long as it needs to provide data to any Subscription.
Since the subscription also contains the closure for the callback that is to be executed, the object that holds the subscription will also remain in memory.
We need to make sure we use a Subscriptions "unsubscribe" method to dispose of the resource.
This component contains an un-disposed subscription. Removing it from the DOM will not remove it from the memory.
