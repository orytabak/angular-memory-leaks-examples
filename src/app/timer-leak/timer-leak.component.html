Leaking timer component
SetInterval/SetTimeout are both methods that belong to the window scope, so calling them from anywhere will create an object accessible from the root scope.
If a component binds a scope when calling them (by providing a bound method or trying to access scope-level variables), or tries storing their references, that component can not be released from memory until the timeout is complete, or the interval is cleared.
To prevent issues, make sure to verify that any timer you do not need is cleared.
Try searching your heap snapshot for "DOMTimer" after closing and reopening this component. You can notice every time this component is created a new DOMTimer object is created as well.
